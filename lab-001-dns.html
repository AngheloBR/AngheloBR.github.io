Lab 001 · Fix DNS en Debian 13

Objetivo: resolver el error Fallo temporal al resolver <deb.debian.org> en un servidor Debian 13 configurando correctamente los DNS.

1. Síntoma

El comando apt update falla al intentar resolver el repo de Debian:

sudo apt update # ... # Fallo temporal al resolver <deb.debian.org>

2. Causa raíz

El archivo /etc/resolv.conf no existía o no tenía servidores DNS configurados, por lo que el sistema no podía resolver nombres de dominio.

3. Remediación aplicada

Crear un /etc/resolv.conf mínimo con DNS públicos:

sudo sh -c 'cat > /etc/resolv.conf << "EOF" nameserver 1.1.1.1 nameserver 8.8.8.8 EOF'

4. Verificación

Probar conectividad y volver a actualizar índices:

ping -c 4 debian.org sudo apt update

En un entorno más serio podrías apuntar a servidores DNS internos (por ejemplo, un resolver de la propia red) en lugar de usar DNS públicos.
