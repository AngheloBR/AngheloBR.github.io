LAB 003 · Security

Hardening Debian 13

Checklist de seguridad para un servidor Debian recién instalado: usuario sudo, SSH seguro, firewall UFW, Fail2Ban y actualizaciones automáticas.

Resumen Checklist de seguridad

  • ✓
    Actualizar el sistema completo
  • ✓
    Crear usuario admin y deshabilitar root
  • ✓
    Configurar SSH seguro (deshabilitar contraseñas)
  • ✓
    Activar firewall UFW con política deny-all
  • ✓
    Instalar y configurar Fail2Ban
  • ✓
    Deshabilitar servicios innecesarios
  • ✓
    Configurar actualizaciones automáticas de seguridad

01 Actualizar el sistema

sudo apt update && sudo apt full-upgrade -y
sudo apt autoremove -y && sudo apt autoclean

02 Crear usuario administrador

# Crear usuario (reemplaza 'admin' con tu nombre)
sudo adduser admin
sudo usermod -aG sudo admin

# Copiar authorized_keys si ya tienes SSH key configurada
sudo mkdir -p /home/admin/.ssh
sudo cp ~/.ssh/authorized_keys /home/admin/.ssh/
sudo chown -R admin:admin /home/admin/.ssh
sudo chmod 700 /home/admin/.ssh
sudo chmod 600 /home/admin/.ssh/authorized_keys
ℹ

Prueba la conexión SSH con el nuevo usuario antes de deshabilitar root: ssh admin@ip_del_servidor

03 Configurar SSH seguro

Edita /etc/ssh/sshd_config:

# Deshabilitar login de root
PermitRootLogin no

# Deshabilitar autenticación por contraseña (solo keys)
PasswordAuthentication no
PubkeyAuthentication yes

# Opciones adicionales de seguridad
X11Forwarding no
MaxAuthTries 3
LoginGraceTime 20
# Aplicar cambios
sudo systemctl restart sshd
⚠

Asegúrate de tener tu clave SSH cargada en ~/.ssh/authorized_keys del usuario admin antes de deshabilitar PasswordAuthentication.

04 Firewall UFW

sudo apt install -y ufw

# Política por defecto: bloquear todo
sudo ufw default deny incoming
sudo ufw default allow outgoing

# Permitir SSH (ajusta el puerto si lo cambiaste)
sudo ufw allow 22/tcp

# Otros servicios según necesites
# sudo ufw allow 80/tcp   # HTTP
# sudo ufw allow 443/tcp  # HTTPS

# Activar firewall
sudo ufw enable
sudo ufw status verbose

05 Fail2Ban

sudo apt install -y fail2ban

# Crear configuración local (no edites el .conf original)
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local

Edita /etc/fail2ban/jail.local en la sección [sshd]:

[sshd]
enabled  = true
port     = 22
maxretry = 3
bantime  = 3600
findtime = 600
sudo systemctl enable --now fail2ban
sudo fail2ban-client status sshd

06 Servicios innecesarios

# Ver puertos abiertos y servicios activos
sudo ss -tulpn

# Deshabilitar servicios que no necesitas
sudo systemctl disable --now <nombre-servicio>

# Ejemplo: deshabilitar avahi si no usas mDNS
sudo systemctl disable --now avahi-daemon

07 Actualizaciones automáticas

sudo apt install -y unattended-upgrades apt-listchanges

# Configurar (solo parchea paquetes de seguridad)
sudo dpkg-reconfigure -plow unattended-upgrades

Verifica la configuración en /etc/apt/apt.conf.d/50unattended-upgrades. Las líneas de seguridad deben estar descomentadas:

Unattended-Upgrade::Allowed-Origins {
    "${distro_id}:${distro_codename}-security";
};
✓

Con esto completado, el servidor tiene una postura de seguridad básica sólida. Para auditorías más profundas, considera instalar Lynis: sudo apt install lynis && sudo lynis audit system
